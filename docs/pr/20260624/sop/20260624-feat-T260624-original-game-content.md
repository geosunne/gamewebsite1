# PR: feat/T260624-original-game-content

## 背景 / 目标
- Bing Webmaster Tools 对部分游戏详情页提示 URL 已知但不可展示，主要风险集中在 iframe 聚合页、批量化标题、第三方描述复用、品牌/版权说明不足。
- 本任务目标是给热门及全部游戏详情页补充更原创的页面说明：玩法、技巧、模式说明、常见问题、设备兼容、加载失败处理，并加入非官方与来源说明。
- 同时减少 `Play X Online Free` 这类批量模板标题，避免 SEO 后处理覆盖生成器的新标题。

## 预检 / 只读盘点
- Problem：游戏详情页 About 文案原先直接使用 `long_description` 或 `description`，重点页如 `shellshockersio` 存在明显第三方描述与家庭评测残留；`optimize_seo.py` 会把标题覆盖为 `Play {title} Online Free - BTW game`。
- Attempts：先用线上 URL、robots、sitemap、Bingbot UA 做过排查，确认不是 200/robots/noindex/canonical 层面的抓取阻塞，更可能是内容质量与策略层面的展示限制。
- Scoped Code Pointers：`generate_static_pages.py::build_game_content`、`build_game_faq`、`build_game_jsonld`、`generate_game_page`；`optimize_seo.py::optimize_game_pages`。
- Open Items：Bing 最终是否恢复展示需等待重新提交和重新抓取；本地只能验证页面结构、内容输出、URL 状态和静态产物一致性。

## 变更摘要
- 新增原创化详情页生成逻辑：基于分类、标签、评分、播放量、控制方式和 iframe 来源生成页面摘要，而不是直接复用第三方长描述。
- 详情页新增可见模块：`Device and browser notes`、`If the game does not load`、`Source and rights note`。
- FAQ 改为覆盖打开前提示、控制方式、游戏类型、加载失败处理、是否官方发布者。
- JSON-LD `Game.description` 改用新 SEO 摘要。
- `optimize_seo.py` 改为使用 `browser guide and quick play notes` 标题与原创摘要，避免二次覆盖回 `Play X Online Free`。
- 重新生成 `static_html/`，产出 1366 个游戏详情页与 1397 个 sitemap URL。

## 影响范围
- 影响静态生成脚本：`generate_static_pages.py`、`optimize_seo.py`。
- 影响生产静态产物：`static_html/games/*.html`、`static_html/all_games.json`、`static_html/sitemap.xml`。
- 不修改数据库 schema、Flask API、后台接口、抓取脚本或 CSS 布局。

## 风险与回滚
- `static_html/games/*.html` 是批量生成产物，diff 很大但来自生成脚本。
- 新文案是规则化生成，不等于人工逐页深度编辑；它降低第三方复用和版权说明风险，但不能保证 Bing 立即恢复索引展示。
- 回滚方式：revert 本分支 merge，或回退 `generate_static_pages.py`、`optimize_seo.py` 后重新生成静态产物。
